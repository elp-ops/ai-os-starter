# Handoffs

One file per project: `handoffs/<project-name>.md`. Written at the end of a session (or when you explicitly ask for one), read automatically at the start of the next one.

This is what lets you say "let's work on X" and have the assistant pick up exactly where things left off, what's done, what's still open, what to do first, without you re-explaining context every time.

A good handoff covers: where the session started, decisions made and what shipped, key files touched, anything still running (background processes, open branches), how to verify things still work, and what's deliberately deferred. See `CLAUDE.md` for the rule that triggers reading these automatically.
